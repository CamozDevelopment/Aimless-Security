Authentication & False Positives

When using authentication endpoints (login, signup, etc.), apply Aimless conditionally to prevent false positives.

Problem: Auth Bypass Detection

Login requests can trigger "auth_bypass_attempt" warnings because they naturally include authentication data. This is a false positive.

‚ö†Ô∏è auth_bypass_attempt (high) - Request blocked

Solution: Conditional Middleware

Skip Aimless security checks for authentication routes while protecting everything else:

const { Aimless } = require('aimless-security');
const aimless = new Aimless({
  rasp: {
    enabled: true,
    blockMode: false,  // Monitor mode recommended during testing
    anomalyDetection: false  // Disable to reduce false positives
  }
});

app.use(aimless.loading());

// Conditional middleware - skip auth routes
app.use((req, res, next) => {
  const authPaths = [
    '/api/auth/login',
    '/api/auth/register',
    '/api/auth/logout',
    '/api/auth/me',
    '/api/auth/profile'
  ];
  
  // Skip Aimless for auth endpoints
  if (authPaths.includes(req.path)) {
    return next();
  }
  
  // Run Aimless for all other routes
  aimless.middleware()(req, res, next);
});
  • Authentication works without blocks
  • All other routes remain protected
  • No false positives on login/signup
  • SQL injection, XSS still detected everywhere

Detects attacks that work as BOTH SQL injection AND XSS simultaneously. These sophisticated payloads exploit multiple vulnerabilities with a single input.

Quote escape + SQL logic + script tag

' OR 1=1--<script>/*XSS*/</script>
SQL: ' OR 1=1-- (SQL comment hides rest)
XSS: <script>/*XSS*/</script> (executes in browser)

XSS: Injected data contains script tag

  console.log(`${vuln.type}: ${score}/100 risk score`);
  console.log(`Severity: ${vuln.severity}`);
  console.log(`Description: ${vuln.description}`);
});

User Management 

const aimless = new Aimless();

app.post('/api/users/profile', (req, res) => {
  const { bio, website } = req.body;
  
  // Validate and sanitize
  const bioResult = aimless.validate(bio)
    .against(['xss', 'sql'])
    .sanitize()
    .result();
    
  if (!bioResult.safe) {
    return res.status(400).json({
      error: 'Invalid bio',
      threats: bioResult.threats
    });
  }
  
  // Safe to save
  db.updateUser({
    bio: bioResult.sanitized,
    website: aimless.sanitizeFor(website, 'url')
  });
  
  res.json({ success: true });
});

API Gateway with IP Blocking 

const aimless = new Aimless();

// Middleware to check IP reputation

  const ip = req.ip;

  if (reputation < 30) {
    return res.status(403).json({
      error: 'IP blocked due to malicious activity'
    });
  }
  
  if (reputation < 50) {
    // Add extra scrutiny for suspicious IPs
    req.extraValidation = true;
  }
  
  next();
});

  res.json(stats);
});
